# Технические задания для разработки программ

## Задача 1. Линейный алгоритм «Перевод времени»

### Техническое задание (ТЗ)

**1. Наименование программы**
`TimeConverter` (Конвертер времени).

**2. Цель создания**
Разработать программу для перевода временного интервала, заданного в часах, в минуты.

**3. Входные данные**
- Одно целое или вещественное число (например, `float`), обозначающее количество часов.
- Диапазон значений: неотрицательные числа (≥ 0).

**4. Выходные данные**
- Одно число (вещественное или целое), обозначающее количество минут.
- Формула для расчета: `минуты = часы * 60`.

**5. Требования к программе**
1.  Программа должна запрашивать у пользователя ввод числа (количество часов).
2.  Программа должна проверять, что введенное значение является числом (защита от букв и спецсимволов, опционально).
3.  Произвести вычисления согласно формуле.
4.  Вывести результат на экран в понятном формате, например: `<часы> ч. = <минуты> мин.`.

**6. Пример работы программы**

> **Ввод:** `2.5`
> **Вывод:** `2.5 ч. = 150.0 мин.`

> **Ввод:** `1`
> **Вывод:** `1 ч. = 60 мин.`

## Задача 2. Алгоритм ветвления «Сравнение чисел»

### Техническое задание (ТЗ)

**1. Наименование программы**
`NumberComparator` (Сравниватель чисел).

**2. Цель создания**
Разработать программу, которая сравнивает два различных числа и определяет, какое из них больше, а какое меньше.

**3. Входные данные**
- Два числа (целые или вещественные).
- Условие: числа должны быть различными (алгоритм должен учитывать этот факт).

**4. Выходные данные**
- Сообщение, информирующее пользователя о соотношении чисел.

**5. Требования к программе**
1.  Запросить у пользователя ввод двух чисел (`a` и `b`).
2.  Сравнить эти числа, используя условный оператор (`if-else`).
3.  Предусмотреть ветку для случая, если числа окажутся равны (хотя по условию задачи они различны, на практике это необходимая проверка целостности).
4.  Вывести одно из следующих сообщений:
    - Если `a > b`: `<число a> больше <числа b>`.
    - Если `a < b`: `<число a> меньше <числа b>`.
    - Если `a == b`: `Числа равны.`

**6. Пример работы программы**

> **Ввод:** `a = 10`, `b = 5`
> **Вывод:** `10 больше 5`

> **Ввод:** `a = 2.5`, `b = 7.1`
> **Вывод:** `2.5 меньше 7.1`

## Задача 3. Алгоритм ветвления «Допуск к экзамену»

### Техническое задание (ТЗ)

**1. Наименование программы**
`ExamAdmission` (Допуск к экзамену).

**2. Цель создания**
Разработать программу для автоматизации проверки допуска студента к экзамену на основе двух критериев: количества пропусков и среднего балла за семестр.

**3. Входные данные**
- Количество пропущенных занятий (целое неотрицательное число).
- Средний балл за семестр (вещественное число, например, по 5-балльной шкале).

**4. Выходные данные**
- Вердикт о допуске или недопуске к экзамену.

**5. Требования к программе**
1.  Запросить у пользователя ввод количества пропусков и среднего балла.
2.  Проверить выполнение **обоих** условий допуска:
    - Пропуски < 10.
    - Средний балл >= 3.0.
3.  Использовать сложное условие (логическое `И` — `&&` или `and`).
4.  Вывести результат на экран.

**6. Пример работы программы**

> **Ввод:** `Пропуски = 5`, `Балл = 4.2`
> **Вывод:** `Студент допущен к экзамену.`

> **Ввод:** `Пропуски = 12`, `Балл = 4.5`
> **Вывод:** `Студент НЕ допущен к экзамену (много пропусков).` (Уточнение причины приветствуется)

> **Ввод:** `Пропуски = 3`, `Балл = 2.7`
> **Вывод:** `Студент НЕ допущен к экзамену (низкий балл).`

## Задача 4. Алгоритм с ветвлением «Скидка в магазине»

### Техническое задание (ТЗ)

**1. Наименование программы**
`DiscountCalculator` (Калькулятор скидок).

**2. Цель создания**
Разработать программу для расчета итоговой стоимости покупки в зависимости от суммы чека: при покупке свыше определенной суммы предоставляется скидка.

**3. Исходные данные (Константы)**
- Порог скидки: 5000 руб.
-


Размер скидки: 10%.

**4. Входные данные**
- Сумма покупки (вещественное положительное число).

**5. Выходные данные**
- Сумма к оплате с учетом скидки (если она применима).
- Информация о примененной скидке.

**6. Требования к программе**
1.  Запросить у пользователя сумму покупки.
2.  Проверить условие: если `сумма > 5000`, то применить скидку 10% (итого = сумма * 0.9).
3.  В противном случае (else) итоговая сумма равна исходной.
4.  Вывести детализацию: исходная сумма, размер скидки (в рублях), итоговая сумма.

**7. Пример работы программы**

> **Ввод:** `6000`
> **Вывод:**
> `Сумма покупки: 6000 руб.`
> `Скидка: 600 руб. (10%)`
> `Итого к оплате: 5400 руб.`

> **Ввод:** `4000`
> **Вывод:**
> `Сумма покупки: 4000 руб.`
> `Скидка не предоставлена.`
> `Итого к оплате: 4000 руб.`

## Задача 5. Алгоритм с вложенными ветвлениями «Квадратное уравнение»

### Техническое задание (ТЗ)

**1. Наименование программы**
`QuadraticEquationSolver` (Решение квадратного уравнения).

**2. Цель создания**
Разработать программу для решения квадратного уравнения вида `ax² + bx + c = 0` с полным анализом всех возможных случаев (в зависимости от дискриминанта и коэффициента `a`).

**3. Входные данные**
- Три вещественных числа: коэффициенты `a`, `b`, `c`.

**4. Выходные данные**
- Корни уравнения или сообщение об отсутствии корней / особых случаях.

**5. Требования к программе (логика работы)**
Программа должна реализовывать следующую логику (алгоритм с вложенными ветвлениями):

1.  **Ввод данных:** Запросить `a`, `b`, `c`.
2.  **Проверка на линейность:** Если `a == 0`, то уравнение становится линейным (`bx + c = 0`).
    - Если `b == 0` и `c == 0`: вывести "Бесконечное множество решений (x - любое число)".
    - Если `b == 0` и `c != 0`: вывести "Нет корней (противоречие)".
    - Если `b != 0`: вывести один корень: `x = -c / b`.
3.  **Основной случай (a ≠ 0):**
    - Вычислить дискриминант: `D = b² - 4ac`.
    - **Ветка 1:** Если `D > 0`: вывести два различных корня:
        - `x₁ = (-b + √D) / (2a)`
        - `x₂ = (-b - √D) / (2a)`
    - **Ветка 2:** Если `D == 0`: вывести один корень (два совпадающих):
        - `x = -b / (2a)`
    - **Ветка 3:** Если `D < 0`: вывести сообщение "Действительных корней нет".

**6. Примеры работы программы**

> **Ввод:** `a = 1, b = -3, c = 2`
> **Вывод:** `Уравнение имеет два корня: x1 = 2.0, x2 = 1.0`

> **Ввод:** `a = 1, b = -4, c = 4`
> **Вывод:** `Уравнение имеет один корень (кратности 2): x = 2.0`

> **Ввод:** `a = 0, b = 5, c = 10`
> **Вывод:** `Линейное уравнение. Корень: x = -2.0`

> **Ввод:** `a = 0, b = 0, c = 5`
> **Вывод:** `Нет корней.`
